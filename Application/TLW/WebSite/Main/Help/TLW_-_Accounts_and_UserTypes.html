Module: Account Database Relationships

 

Purpose: Describes how a user is in an account, and their user type and if they are the ‘Main’ holder.

 

‘Main’ vs. ‘Extended’

  • How Users connect to a single participant.
  • USER_INFO.ACCOUNT_FROM is the USER_ID of the ‘Main’ Account
    • Example: John Pretzel is a ‘Main’ account holder because he works for Pretzels. And Johnny Jr is an ‘Extended’ member as John’s dependant.
      • If(USER_INFO.USER_ID == USER_INFO.ACCOUNT_FROM)
        • ‘Main’
      • Else
        • ‘Extended’
    • Note: ACCOUNT_FROM_RELATIONSHIP_ID links in LIST_ITEMS, and stores the relationship. IE Spouse, Dependant, etc.

               

 

 

 

 

 

 

 

 

 

Accounts and Users

  • User’s USER_TYPE_ID (Currently)
    • When a user is placed in an account, he is given a USER_TYPE_ID
      • 0 = None
      • 1 = Manager
        • Can see reports
      • 2 = User
