TLW - User Registration Process

 

Setting up an account

 

This system is designed to allow an Administrator to collect data for main users (ie Member) and then create those users, and sub-accounts for those users (ie Spouse, Domestic Partner, Child/Dependent). 

 

Special requested fields for the user are stored in USER_INFO and are currently as follows:

 

  • GENDER_ID, FAMILY_STATUS_ID, and ACCOUNT_FROM_RELATIONSHIP link to table LIST_ITEMS.
  • DATE_BIRTH - birthday of the user.
  • REGISTRATION_FLG – 1= User is registered. 0= false.   
    • Note: they can’t login if they haven’t completed registration.
    • The main account holder should be marked as registered when created.
  • ACCOUNT_CREDITS – No longer used.
  • ACCOUNT_FROM – The USER_ID of the user who is the main account holder. So main user John Smith has his ACCOUNT_FROM=USER_ID.  While Johnny Jr. Has his ACCOUNT_FROM= John Smith’s USER_ID.

Activating sub-accounts

 

To complete registration for a sub-account, the main account holder simply needs to select which sub-account they would like to activate, and update that information.  When the main account holder has sub-accounts that haven’t completed registration, they will be shown their “Account Credits,” which they can click on and then select a sub-account from the list provided.  After that, they need only to provide information for the sub-account, and click Update.

 

 

 

 

Once registration is complete for the sub-account, they may log in and begin using the website as normal.
